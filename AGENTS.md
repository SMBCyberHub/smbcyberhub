## SEO/AEO Quality Gate

**Before publishing any new or updated content, verify it meets or exceeds the SEO/AEO standards established across the existing site.** Read `memory-bank/systemPatterns.md` to understand the full pattern set, then check every item below.

### Structured Data (GSC will flag violations)
- BlogPosting schema auto-generated by `[slug].astro` — confirm frontmatter provides: title, description, date, dateModified, ogImage, excerpt, tags, canonical
- No duplicate JSON-LD blocks on the same page
- No FAQPage schema on noindex pages
- All schema URLs must use trailing slashes and match the canonical URL
- SpeakableSpecification present (auto-generated for blog posts; manually added on static pages)

### Indexing & Crawl
- `robots` meta must be `index,follow` on all content pages (default; only terms/privacy/licensing are noindex)
- Canonical URL must be consistent between frontmatter, Layout, and schema
- New/changed pages must appear in sitemap — check `astro.config.mjs` staticDates for static pages, and `dateModified` frontmatter for blog posts
- New blog posts must have a trailing-slash 301 redirect added to `public/_redirects`

### Open Graph & Social
- Blog posts must pass `ogType="article"` to Layout (handled by template)
- `article:published_time`, `article:modified_time`, and `article:tag` must render correctly from frontmatter
- `og:locale` must be `en_IE` with `en_US` alternate (handled by Layout)

### Content Quality / E-E-A-T
- All posts must have proper author attribution (hardcoded in template as Jim SMBCyberHub)
- `datePublished` and `dateModified` must both be present and accurate in frontmatter
- No thin content — blog posts should exceed 1,500 words with clear H2/H3 hierarchy
- No duplicate content between posts — cross-link instead of repeating

### AEO (AI Engine Optimization)
- Use question-as-heading + immediate answer paragraph pattern (AI engines extract these)
- Maintain clear H2/H3 topic hierarchy for segmentation
- RSS feed at `/rss.xml` must include full-text content (auto-handled by `rss.xml.js`)
- SpeakableSpecification must target `article h1` and first paragraph

### Internal Linking
- New posts must NOT be orphan pages — add inbound links from at least 2 existing related posts
- New posts must link OUT to at least 3 existing pages with descriptive anchor text (not "click here")
- All internal link hrefs must use trailing slashes

### Metadata Limits
- Title tags: <= 60 characters (including " | SMBCyberHub" suffix)
- Meta descriptions: 130-160 characters (not too short, not over limit)
- og:description should match or be derived from meta description

### Post-Publish Checklist
After any content change, before committing:
1. Run `npx astro build` — must be 0 errors
2. Verify new post appears on `/blog/` in correct category
3. Verify `featured: true` posts appear in Featured section if intended
4. Confirm sitemap lastmod dates are current for all changed pages
5. List URLs to submit to Google Search Console for re-indexing

## Content Rules

- **SEO/AEO is the only marketing channel.** Every content and technical decision must prioritise search/AI discoverability.
- **Do NOT claim the kit helps pass Cyber Essentials.** CE tests technical controls. The kit covers policies, training, and documentation (complementary layer).
- **Password guidance must be NCSC-aligned.** Three random words, 12+ chars (or 8 with deny list). No forced expiry, no complexity rules.
- **All blog post titles must be <= 60 characters** (including " | SMBCyberHub" suffix added by template).
- **All meta descriptions must be <= 160 characters.**
- **All internal links must use trailing slashes.**
- **New blog posts need a trailing-slash redirect** added to `public/_redirects`.
